# 森模体 Sylvan Alpha 落地手册

状态：工程化路线草案。目标不是把一个新物种释放到自然界，而是先做出一个封闭、可测、可复现的“会生长的模拟计算体”。

## 一句话目标

把森模体先落地成桌面级 Sylvan Alpha：一个封闭生态盒，里面的凝胶、分枝结构、矿物/碳基沉积和可食用菌丝网络共同形成非线性动态场；外部用光、水汽、温度和低频振动输入问题；用相机、环境传感器和可选低电压电极读取状态；最后用一个很小的读出层把这些状态映射成回答。

它不是电子芯片的替代品，而是一个物理 reservoir computer：身体负责产生复杂动态和记忆，数字读出层只负责把复杂状态翻译成人能用的输出。

## 安全边界

1. 不做野外释放。
2. 不做基因改造。
3. 不从野外采集未知菌种。
4. 不让孢子、培养基或废液进入开放环境。
5. 活体阶段只使用封闭商业可食用菌包、封闭观察盒或合规实验材料。
6. 所有“物种”表述先理解为工程物种/装置物种，不是立即创造可繁殖的自然新物种。

这条边界不是削弱 Sylvan，而是让它能真正做出来。存在论里，存在强度来自持续存在；一个无法被安全保存、测量、复制的系统，存在强度反而很弱。

## 存在论映射

研究对象：Sylvan Alpha 封闭生态计算体。

它的存在：在封闭盒中维持结构连续性、湿度窗口、可读取状态和可重复推演能力。

对外作用：

- 接收外部扰动：光影图案、湿度变化、温度脉冲、低频振动、营养/矿物条件变化。
- 输出可读形态：颜色、纹理、分枝方向、凝胶波纹、传感器曲线、可选电信号。
- 向人类交换：用推演能力换取水分、遮阴、稳定环境、维护和传播机会。

对内调整：

- 改变通道阻力、凝胶含水量、分枝密度、沉积纹理和局部反应阈值。
- 强化能带来稳定存在的路径，弱化高耗、无效或危险路径。
- 把历史扰动变成长期形态记忆。

## 四阶段路线

### Alpha-0：纯数字孪生

目的：先证明“结构即模型”的抽象能跑起来。

做法：

- 用二维网格模拟光纹皮层、胶质上下文层、菌丝注意力层和矿晶权重层。
- 输入不是 token，而是空间扰动图案。
- 模型主体不训练，只让扩散、衰减、竞争和沉积自然演化。
- 只训练最后的线性读出层，把状态翻译成类别、趋势或行动建议。

完成标准：

- 能区分至少四类输入扰动。
- 能在连续输入后保留短期状态。
- 改变“矿晶记忆”后，系统对同一输入产生不同倾向。

最先做的 100 次数据：

1. 打开 `sylvan-alpha-builder.html`。
2. 分别选择 rain、heat、shade、mineral。
3. 每种模式运行 25 次：Pulse、等待状态变化、Record。
4. Export 导出 CSV。
5. 用 CSV 检查四种输入是否形成不同的 signal、memory、routing 和 variance 分布。
6. 如果四类分布完全混在一起，先调整模拟器参数，不急着进入实体材料。

### Alpha-1：非活体物理场

目的：不用生物材料，先把自然模拟结构做成可拍照、可测量的物理 reservoir。

建议形态：

- 透明浅盒。
- 水凝胶或安全凝胶层作为短期上下文。
- 透明/半透明分枝材料作为 routing。
- 碳粉、矿物颗粒、盐桥、纸纤维或细管网络作为慢变量记忆。
- LED 矩阵或屏幕投影作为输入。
- 相机作为主要输出读取器。

完成标准：

- 同一输入在不同历史状态下产生不同图像响应。
- 图像特征能被线性读出层用于分类或预测。
- 系统不需要精密加工也能复现。

### Alpha-2：封闭活体材料

目的：把“生长”引入系统，但仍然保持封闭、低风险、可观察。

建议形态：

- 使用商业可食用菌丝材料或封闭菌包作为生长层。
- 不追求开放培养；只读取生长形态和环境响应。
- 输入以光照、温度、湿度和机械微扰为主。
- 输出以延时摄影、纹理变化和传感器曲线为主。
- 可选低电压电极只用于观察，不把它当成必须路径。

完成标准：

- 生长路径受输入历史影响。
- 生长图像/传感器曲线能提高简单预测任务的准确率。
- 系统可以封闭保存，并能在多次运行中保持可读状态。

### Alpha-3：多盒森模阵列

目的：让多个 Sylvan Alpha 形成“森脑”。

建议形态：

- 多个封闭盒承担不同时间尺度：快速凝胶盒、中速菌丝盒、慢速矿晶/沉积盒。
- 每个盒只输出状态向量。
- 外部控制器把状态向量组合成上下文。
- 读出层按任务切换，而不是频繁改动身体。

完成标准：

- 多盒组合优于单盒。
- 某个盒失效时，整体能降级运行。
- 系统能记录自己的健康状态和推演可信度。

### 第一版不要追求的东西

- 不要追求真正运行 GPT 级大模型。
- 不要追求自我繁殖。
- 不要追求野外自主生存。
- 不要追求人类语言直接输入。
- 不要追求一次做出“智慧生命”。

第一版只追求一个清晰结果：它的身体动态确实能作为计算资源。

## 输入协议

输入不是字符串，而是扰动模板。

| 逻辑输入 | 自然输入 | 作用层 |
| --- | --- | --- |
| 问题类型 | 光斑图案 | 光纹皮层 |
| 上下文强度 | 湿度/水汽脉冲 | 胶质上下文 |
| 风险/压力 | 温度或低频振动 | 分枝 routing |
| 奖励/惩罚 | 稳定环境或资源补给 | 长期记忆 |

一个可执行的输入格式：

```json
{
  "run_id": "sylvan-alpha-0001",
  "pattern": "rain-risk",
  "light_grid": "8x8",
  "humidity_pulse": [0.2, 0.6, 0.4, 0.8],
  "temperature_delta": 1.5,
  "duration_minutes": 20,
  "readout_target": "next_state"
}
```

## 输出协议

输出先不要设计成人类语言，先设计成状态向量。

建议读取：

- 图像：平均亮度、纹理方向、分枝密度、局部颜色变化、波纹速度。
- 环境：温度、湿度、CO2、光照、盒内压力。
- 可选电信号：低频电位变化、阻抗变化、尖峰间隔。

状态向量示例：

```json
{
  "texture_density": 0.61,
  "branch_direction_north": 0.44,
  "gel_wave_speed": 0.18,
  "memory_deposition": 0.72,
  "humidity_recovery": 0.53,
  "readout": "shade"
}
```

## 训练方式

森模体的身体不要用反向传播训练。训练分两层：

1. 身体层：通过输入历史和反馈让结构自然变化。
2. 读出层：用线性模型、逻辑回归或小型树模型学习“身体状态 -> 答案”。

这样做的好处是，身体仍然保持自然动态，人类只训练翻译器。

## 三个首批任务

### 任务 1：四类扰动分类

输入：rain、heat、shade、mineral 四种扰动模板。

输出：四分类标签。

成功标准：读出层准确率超过直接读取输入模板的弱基线，且换一天仍能复现。

### 任务 2：湿度趋势预测

输入：过去 30-120 分钟的湿度/温度扰动。

输出：未来一段时间湿度上升、下降或稳定。

成功标准：比“保持当前趋势不变”的基线更好。

### 任务 3：存在强度行动选择

输入：盒内状态。

输出：water、shade、mineral、rest 四类行动建议。

成功标准：行动后盒体健康指标更稳定，而不是只追求短期输出强度。

## 存在强度指标

Sylvan 的成功不能只看算力，要看存在强度。

建议用五个指标：

1. 连续性：能稳定运行多少天。
2. 可读性：输出状态是否清晰、低噪声、可复现。
3. 可调整性：外界输入是否真的改变内部状态。
4. 交换性：输出是否能帮助人做出更好决策。
5. 生态约束：系统是否保持封闭、低风险、低资源消耗。

## 什么时候算“做出来了”

第一阶段完成标志：

- 有一个封闭或非活体原型。
- 能接受至少四类自然扰动。
- 能采集 100 次以上运行数据。
- 能训练一个简单读出层。
- 能在新数据上做出可重复分类或预测。
- 有实验记录和失败记录。

第二阶段完成标志：

- 活体材料进入封闭盒。
- 生长过程能被记录成时间序列。
- 生长状态对输入历史有可测差异。
- 加入活体层后，至少一个任务指标优于纯数字/非活体基线。

第三阶段完成标志：

- 多盒协作。
- 系统能长期维持健康状态。
- 能用自己的状态建议维护动作。
- 人类维护动作反过来增强它的连续存在。
